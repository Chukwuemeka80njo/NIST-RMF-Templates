# USER ACCESS REQUEST PROCEDURE  
Version: 1.0  
Approval Date: (insert date)  
Owner: Information Security Officer (ISO)  
Classification: Internal / Restricted  

---

## **1. Purpose**  
The purpose of this procedure is to define the steps required to request, approve, create, modify, or remove access to organizational information systems.

---

## **2. Scope**  
This procedure applies to:  
- All employees, contractors, and third parties requiring system access  
- All organizational information systems, applications, and platforms  
- All privileged and non-privileged user accounts  

---

## **3. Prerequisites**  
- A valid business need must be established  
- The requester’s identity must be verified  
- Required approvals must be obtained before access is provisioned  

---

## **4. Procedure Steps**

### **4.1 Access Request Submission**  
1. User submits a formal **Access Request Form** (ARF) or ticket.  
2. Form must include:  
   - Requester’s full name  
   - Job title and department  
   - System(s) requested  
   - Type of access (read-only, admin, privileged, etc.)  
   - Business justification  

---

### **4.2 Manager Review & Approval**  
1. Request is routed to the user’s manager.  
2. Manager validates:  
   - The business need is legitimate  
   - Access level requested is appropriate  
3. Manager approves or denies the request.  
4. Denials must include justification.

---

### **4.3 ISO / Security Review (for privileged accounts)**  
Privileged, administrative, or sensitive-access requests require ISO or delegated security approval.  
Security verifies:  
- User has required training  
- Least privilege is enforced  
- No conflicts of interest  
- MFA is enabled  

---

### **4.4 Provisioning Access**  
1. IT provisioner creates account or updates existing account.  
2. All access must follow:  
   - **Least privilege**  
   - **Separation of duties**  
   - **Role-based access control (RBAC)**  
3. Provisioner documents:  
   - Username  
   - Access level granted  
   - Date provisioned  
   - Ticket or ARF reference  

---

### **4.5 Notification**  
IT notifies the requester and manager when access is provisioned.  
User is provided with:  
- Login instructions  
- Password reset instructions  
- Security expectations  

---

### **4.6 Documentation & Recordkeeping**  
- All access requests must be stored for **a minimum of 1 year**  
- Approval records must be retained  
- System access logs must reflect provisioning activity  
- The Access Control List (ACL) must be updated  

---

## **5. Access Modification Requests**  
Modifications follow the same workflow as new requests and must include:  
- Reason for modification  
- Changes needed  
- Manager approval  
- Security approval for privileged modifications  

---

## **6. Access Removal / Revocation**  
Access must be revoked under the following conditions:  
- Employee termination (immediate)  
- Role or department change  
- Contractor engagement ending  
- Security concerns or policy violations  

Revocation steps:  
1. Manager submits a removal request or HR automation triggers it.  
2. IT disables all accounts within **24 hours** (or immediately for terminations).  
3. Documentation is updated.

---

## **7. Review & Audit**  
- Access rights must be reviewed **quarterly**  
- Privileged accounts must be reviewed **monthly**  
- Any discrepancies must be corrected within 5 business days  

---

## **8. Responsibilities**  
- **Requester:** Submit accurate access request  
- **Manager:** Validate business need and approve  
- **Security/ISO:** Review privileged access  
- **IT:** Provision, modify, or remove access  
- **HR:** Notify changes in employment status  

---
