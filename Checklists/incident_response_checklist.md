# Incident Response Checklist

> Use this checklist to ensure effective preparation, detection, response, and recovery for security incidents.

---

## 1. Preparation
- [ ] Incident Response Plan approved and accessible  
- [ ] IR team roles assigned and trained  
- [ ] Contact lists updated (internal + external)  
- [ ] Tools and forensic utilities available  
- [ ] Backup and restoration procedures tested  
- [ ] Communication protocols defined  

---

## 2. Detection & Analysis
- [ ] Alerts monitored (SIEM, IDS/IPS, AV, logs)  
- [ ] Suspicious activity properly triaged  
- [ ] Incident severity level assigned  
- [ ] Evidence collected and preserved  
- [ ] Initial scope and impact assessed  

---

## 3. Containment
- [ ] Affected systems isolated  
- [ ] Temporary controls implemented  
- [ ] Malicious accounts or processes disabled  
- [ ] Network containment steps executed  
- [ ] Short-term and long-term containment reviewed  

---

## 4. Eradication
- [ ] Root cause identified  
- [ ] Malware removed  
- [ ] Vulnerabilities patched  
- [ ] Unauthorized changes reverted  
- [ ] Indicators of compromise (IOCs) removed  

---

## 5. Recovery
- [ ] Clean systems restored  
- [ ] Services brought back online safely  
- [ ] Monitoring increased post-recovery  
- [ ] Users notified if applicable  

---

## 6. Post-Incident Review
- [ ] Lessons learned meeting completed  
- [ ] Timeline of incident documented  
- [ ] IR Plan updated  
- [ ] Preventative measures implemented
